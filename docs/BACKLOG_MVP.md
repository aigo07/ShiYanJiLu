## Backlog：客户调样合成实验记录系统（MVP）

### 说明
- **P0**：没有它就无法上线/无法日常使用
- **P1**：显著提升效率与数据质量（建议紧跟P0）
- **P2**：体验/扩展项（后续迭代）

---

## P0（必须）

### P0-1 工程骨架与本地启动（前后端+数据库）
- **交付**：项目可在本机一键启动（开发模式），包含前端、后端、数据库（推荐 PostgreSQL）。
- **验收**：
  - 新人按 README 操作可在 30 分钟内跑起来
  - 后端健康检查接口返回 OK
  - 前端能访问到后端（最小 API 调通）
- **依赖/备注**：无

### P0-2 数据库 Schema：Experiment / Record / CuringAgent（含“未知”）
- **交付**：
  - `experiments`、`records`、`curing_agents`（以及后续可能的版本表预留）
  - seed 导入硫化剂字典，且强制包含 `“未知”`
- **验收**：
  - 数据库迁移可重复执行（空库可建表）
  - seed 执行后 `curing_agents` 中必定存在 `未知`
  - 删除数据库后重建+seed，结果一致
- **依赖/备注**：种子数据来源 `raw_data/catalysts.json`

### P0-3 后端 API：Experiment CRUD
- **交付**：创建/查询/更新/删除 Experiment 的 API。
- **验收**：
  - 可创建 Experiment，必填字段缺失返回 4xx + 明确错误
  - 列表查询支持按 `customer_name/project_no/silicone_model/time_range` 过滤（至少 2 个条件）
  - 删除 Experiment 时关联的 Record 处理策略明确（推荐：级联删除或软删除）
- **依赖/备注**：依赖 P0-2

### P0-4 后端 API：Record CRUD + 强校验（防前端绕过）
- **交付**：在某个 Experiment 下新增/修改/删除 Record 的 API，并在后端做强校验：
  - `curing_agent_a_id`、`curing_agent_b_id` 必填（未知时用“未知”）
  - `ratio_a_pct`、`ratio_b_pct` 取值 0–100，且 `A+B<100`
  - `bubble_grade` 为整数 0–5
  - `t10/t90` 解析后存为 `t10_sec/t90_sec`（整数秒；可配置上限 ≤300）
- **验收**：
  - 直接用接口提交非法比例（如 A=60,B=50）会被拒绝（4xx）
  - 提交 `t90="0:53"` 能成功并存成 `53`
  - 提交无法解析的 `t10` 返回 4xx 并提示输入格式
- **依赖/备注**：依赖 P0-2；T10/T90 解析规则需单测覆盖

### P0-5 前端页面：Experiment 列表页
- **交付**：
  - 展示 Experiment 列表（客户/项目编号/胶型号/目标/起止时间/Record 数）
  - 支持新建、进入详情、删除
- **验收**：
  - 可按客户/胶型号关键字搜索
  - 列表加载与空状态展示正确
- **依赖/备注**：依赖 P0-3

### P0-6 前端页面：Experiment 详情页（meta+默认工艺）+ 保存
- **交付**：Experiment 详情页可编辑 meta 与默认工艺字段并保存。
- **验收**：
  - 必填字段未填会有明确提示且不可保存
  - 保存成功后刷新仍能看到最新值
- **依赖/备注**：依赖 P0-3、P0-5

### P0-7 前端页面：Record 表格（新增/编辑/删除）+ “复制上一条”
- **交付**：
  - 在 Experiment 详情页下半区展示 Record 表格
  - 支持新增/编辑/删除 Record
  - 支持“一键复制上一条 Record”（带出硫化剂/比例/工艺覆盖等，用户再改差异）
- **验收**：
  - 能连续录入 10 条 Record，操作成本低（复制可用）
  - 录入比例/气泡/T10/T90 的校验提示清晰
- **依赖/备注**：依赖 P0-4；硫化剂下拉依赖 P0-8

### P0-8 前端：硫化剂下拉数据源（含“未知”）+ 基础字典页（只读也可）
- **交付**：
  - 前端能拉取 `curing_agents` 作为下拉候选
  - 可选：提供硫化剂列表页（MVP 可只读，先保证可用）
- **验收**：
  - 下拉中始终存在 `未知`
  - 当 seed 更新后刷新可看到新增硫化剂
- **依赖/备注**：依赖 P0-2（seed）

### P0-9 前端：客户与胶型号“历史选择 + 可新建”
- **交付**：
  - 在 Experiment 创建/编辑时，`customer_name` 与 `silicone_model` 支持下拉搜索历史值
  - 不存在时可直接输入并保存为新候选（或自动进入历史）
- **验收**：
  - 新建一个从未出现的客户名，保存后再次新建 Experiment 时可在下拉中搜索到
  - 胶型号同理
- **依赖/备注**：需要后端提供“distinct 值”接口或独立维表（实现方案由研发决定）

### P0-10 Record 检索页（事实表）+ CSV 导出
- **交付**：
  - Record 检索页：按客户/项目/胶型号/目标、硫化剂A/B、ML/MH/T10/T90区间、气泡等级、工艺区间过滤
  - 一键导出当前筛选结果为 CSV
- **验收**：
  - 筛选条件组合生效且结果正确（至少覆盖 5 个组合用例）
  - CSV 字段固定，包含 Experiment 关键 meta + Record 字段
- **依赖/备注**：后端需要可分页查询与导出接口（可同一套过滤参数）

## P1（强烈建议）

### P1-1 Record 工艺覆盖（UI 体验优化）
- **交付**：Record 编辑时支持折叠区覆盖工艺字段；默认继承 Experiment 值并清晰展示“继承/覆盖”状态。
- **验收**：
  - 不填覆盖时，列表/详情显示继承值
  - 填了覆盖时，仅该 Record 使用覆盖值
- **依赖/备注**：依赖 P0-7；后端字段已预留

### P1-2 字典页：硫化剂 CRUD（最小可用）
- **交付**：硫化剂列表/新增/编辑/停用（配方条目可后置）。
- **验收**：
  - 新增硫化剂后，Record 下拉可选到
  - 停用的硫化剂默认不出现在下拉（或标记为停用）
- **依赖/备注**：与版本化策略可先简化

### P1-3 T10/T90 解析器单测 + 边界用例
- **交付**：覆盖输入格式：`65`、`0:53`、`00:53`、`53s`、`1m5s`、非法输入。
- **验收**：
  - 单测覆盖率达成（以你们工具链为准）
  - 非法输入返回一致的错误码与提示
- **依赖/备注**：依赖 P0-4

### P1-4 删除/编辑审计（最小审计）
- **交付**：为 Experiment/Record 增加 `created_at/updated_at`；删除时记录删除时间（软删或日志二选一）。
- **验收**：
  - 能追溯一条 Record 最近更新时间
  - 删除不会“静默消失”而无法追踪（至少留痕）
- **依赖/备注**：单用户也建议做

## P2（后续）

### P2-1 文件附件上传（图片/仪器导出文件）
- **交付**：在 Experiment 与 Record 支持上传并查看附件。
- **验收**：
  - 上传后可预览/下载
  - 删除 Experiment 时附件处理策略明确

### P2-2 更丰富的可视化（趋势/分布/相关）
- **交付**：在检索结果上提供简单图表（T90 分布、ML/MH 散点等）。
- **验收**：
  - 图表与筛选条件联动

### P2-3 配方版本化严格落地
- **交付**：硫化剂配方版本表 + Record 绑定版本；历史版本只读。
- **验收**：
  - 修改硫化剂只能生成新版本
  - 老 Record 引用的版本不可变
